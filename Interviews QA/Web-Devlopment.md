5. what is MVT?

Ans:Django follows the model-view-template.
Model:Defines the data structure (using Python classes that interact with the database via the ORM).

View:Contains the logic that processes user requests, interacts with the model to fetch or modify data, and decides what data to send to the template.

Template:A text file (usually HTML) with Django-specific tags and logic that defines how the data should be presented to the user in the browser.

9. what is protocol?
Ans: A protocol is a standardized set of rules that define how data is formatted, transmitted, and interpreted during communication between devices or systems.

 

10. what is http?
Ans: HTTP stands for Hypertext Transfer Protocol and is the foundational protocol for data communication on the World Wide Web.

11. what is https?
Ans:HTTPS stands for HyperText Transfer Protocol Secure. It is the secure version of the standard HTTP, the fundamental protocol for sending data between a web browser and a website. 

 

12. what is difference between http and https?
Ans:The main difference is that HTTPS is a secure version of HTTP. HTTPS encrypts data using SSL/TLS certificates, protecting it from being intercepted and read, while HTTP transmits data in plain text and is not secure.

13. what is cors in web api?
Ans:Cross-origin resource sharing (CORS) is a mechanism for integrating applications. CORS defines a way for client web applications that are loaded in one domain to interact with resources in a different domain.

14. what is URL?

Ans:A URL (Uniform Resource Locator) is the address of a web page or resource on the internet, similar to a street address that guides a web browser to find specific information online. 

15. what is JSON?

Ans:JSON, which stands for JavaScript Object Notation, is a lightweight and text-based data format for storing and exchanging data. JSON is commonly used in web applications for sending data between a server and a client, and is supported by many different programming languages and systems.
